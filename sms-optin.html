VerbaVox SMS Opt-In Flow

How users consent to receive SMS verification messages

Compliance Summary

  • Users voluntarily enter their phone number in the mobile app
  • Clear disclosure is shown BEFORE user takes action
  • User must tap "Send Verification Code" to opt-in
  • No messages sent without explicit user action
  • Message frequency and data rates disclosed
  • Opt-out instructions provided (Reply STOP)
1
User Downloads App & Starts Registration

User downloads VerbaVox from App Store or Google Play and opens the app to create an account.

2
User Enters Phone Number

User voluntarily enters their mobile phone number for account verification.

User Taps "Send Verification Code" (Explicit Opt-In)

Only after the user reads the disclosure and taps the button, we send ONE SMS message with a verification code. This is the explicit call-to-action (CTA) that triggers the opt-in.

Key Compliance Points

  • Disclosure shown BEFORE opt-in: User sees consent text before tapping the button
  • Single opt-in action: Tapping "Send Verification Code" is the explicit CTA
  • No pre-checked boxes: User must actively tap to consent
  • Message frequency stated: "1 message per request"
  • Rates disclosure: "Message and data rates may apply"
  • Opt-out method: "Reply STOP to opt-out"
  • Privacy Policy linked: Accessible from the consent screen
  • No marketing messages: Only transactional verification codes
